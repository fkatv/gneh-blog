Objetivo: Entender y crear un componente estilizado con Tailwind y el sistema de diseño Tailwind Elements en Angular.

Al definir los estilos de los elementos de una página, es frecuente ver extensas líneas horizontales con clases de Tailwind, que a menudo, se expanden según las necesidades de diseño que se desee obtener en un elemento de página. Cuando este elemento estilizado se usa en varias partes del proyecto, genera demasiados bytes de código repetido. Veremos cómo optimizar y limpiar el código fuente de las repetitivas configuraciones visuales con el uso de componentes contextuales.

Dile hola a los: ReButtons!

Este fragmento de código estiliza los botones que están arriba. Imagínate que sin esta estrategia hubiera repetido seis veces el código

Qué, porqué, para qué y cómo hacer un ReButtons!

Qué es?

Un componente contextual estilizado de un botón en Angular. Lo llamo contextual porque recibe el contexto del padre como parámetro para que al traspasar las funciones (que podrían manejar variables de la clase padre) no se pierda el contexto this original y evitar así usar los ng-templates. Lo veremos con detalle más adelante.

Porqué hacer este componente?

Sabiendo que hay varias herramientas de diseño gratuitas y de pago que traen plantillas listas para usar, la idea es aprender a usar componentes reutilizables en Angular, entender la estrategia contextual, aprender a usar el ng-class y simplificar la complejidad de usar templates (al menos, para botones simples)

Para qué hacer este componente?

Para mejorar tus habilidades de frontend, para crear nuevas plantillas o sistemas de diseño en Angular, mejorar la lectura (readability), reutilización de código y para crear tus propios diseños controlados por tu propio componente, qué mejor!
