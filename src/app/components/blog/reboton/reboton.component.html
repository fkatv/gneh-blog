Objetivo: Entender y crear un componente estilizado con Tailwind y el sistema de diseño Tailwind Elements en Angular.

Al definir los estilos de los elementos de una página, es frecuente ver extensas líneas horizontales con clases de Tailwind, que a menudo, se expanden según las necesidades de diseño que se desee obtener sobre un elemento de página. Al trabajar sin Tailwind CSS, el estilizado se realiza configurando una hoja de estilos CSS cuyas configuraciones se conectan con la página web, o bien, se emplea la configuración explícita de estilos sobre el mismo elemento HTML, esto implica la dependencia de la hoja de estilos al componente o emplear estilos inyectados en alguna sección del código. Sumado a lo anterior, cuando se necesita reutilizar elementos estilizados en el proyecto, aumenta necesariamente las líneas de código repetido si no se maneja adecuadamente. Veremos en esta entrada, cómo optimizar y limpiar el código fuente de las repetitivas configuraciones visuales con el uso de componentes contextuales.

Entendiendo algunas estrategias de diseño

En la práctica,

Tailwind cumple su labor de ayudar al diseño y convertir los elementos HTML en candidatos a componentes según donde sea usado, preferentemente en frameworks de páginas web basados en javascript como Angular, React, Vue, etc.

Imagínate ahora, un componente que no solo luzca bien, sino que también, interaccione con el mouse y que además, sea afectado por eventos de otros componentes considerando si el usuario cometió un error en algún área o se afecta por los datos que recibe de alguna petición HTTP, entre otros eventos, consecuentemente, crece la cantidad de clases que afectan al diseño dinámico de este elemento. A todo lo anterior, agrégale que este elemento lo usarás en varias secciones y páginas del proyecto... tendrás que copiar todo el código afectando en la escalabilidad y calidad de desarrollo. Una estrategia para abordar esta situación es usar componentes, así como se ve en la siguiente imágen comparativa.

Y si uso una clase global en vez de Tailwind CSS?

Es posible, sin embargo, para llegar a ese nivel de abstracción se debe tener profundamente dominado el diseño y sus interacciones. En etapas de diseño, los elementos están en constante modificación según la necesidad de diseño e interacción que se desee, es por eso que, usar clases globales, podría ser viable en etapas más maduras del proyecto. Mientras se diseña, la visual debe ser altamente moldeable, escalable y prototipable, por lo que emplear componentes , es una vía al desarrollo contínuo e iterativo de elementos que tienden a usarse en varias secciones del proyecto de software. Presentación del componente ReButtons!

Qué es?

Un componente contextual estilizado de un botón en Angular. Lo llamo contextual porque recibe el contexto del padre como parámetro para que al traspasar las funciones (que podrían manejar variables de la clase padre) no se pierda el contexto this original y evitar así usar los ng-templates. Lo veremos con detalle más adelante.

Porqué hacer este componente?

Sabiendo que hay varias herramientas de diseño gratuitas y de pago que traen plantillas listas para usar, la idea es aprender a usar componentes reutilizables en Angular, entender la estrategia contextual, aprender a usar el ng-class y simplificar la complejidad de usar templates (al menos, para botones simples)

Porqué emplear Tailwind Elements?

Usar un sistema base de diseño mejora los tiempos de desarrollo, como también se puede utilizar maquetas prehechas, gratuitas y altamente escalables. Tailwind Elements tiene mas de 500 componentes para utilizar, lo que resulta perfecto para crear componentes personalizados.

Para qué hacer este componente?

Para mejorar tus habilidades de frontend, para crear nuevas plantillas o sistemas de diseño en Angular, mejorar la lectura (readability), reutilización de código y para crear tus propios diseños controlados por tu propio componente, qué mejor!

Cómo hacer el componente contextual desde cero?

Roadmap

  • Crea tu proyecto Angular
  • Instala y configura Tailwind & Tailwind Elements
  • Crea e integra un módulo de Angular
  • Crea un componente (ReButtons) en el módulo
  • Invoca el componente (ReButtons) en tu HTML
  • Agrégale una función y un contexto (opcional)
  • Agrégale íconos, imágenes al componente (opcional)

Manos a la obra!

1. Crea tu proyecto Angular

npm install -g @angular/cli
ng new my_project && cd my_project

2. Instala y configura Tailwind & Tailwind Elements

Basado en el sitio oficial de TW Elements, recomienda:

Node LTS (18.x.x or higher recommended)
Angular (14.x.x or higher recommended)
Code editor. We recommend VSCode

Seguir instrucciones acá.

3. Crea e integra un módulo de Angular

Construyamos un módulo visual llamado 'ux', que contendrá todos los componentes que querramos construir y reusar. Podrías pensar que éste módulo es tu propio sistema de diseño!. Lo generaremos tipeando lo siguiente:

ng g m ux

Ahora integraremos este nuevo módulo al manejador de módulos de Angular del proyecto: app.module.ts. Si te fijas en la imágen siguiente, dentro del NgModule, se importa nuestro módulo UxModule, con esto, podemos emplear todos sus componentes dentro de cualquier lugar del proyecto.

4. Crea el componente (ReButtons) en el módulo

Generaremos el componente dentro del módulo tipeando lo siguiente

ng g c ux/rebutton

Paso a seguir es definir un parámetro de entrada que configurará el comportamiento y visualización de este botón dinámico, para ello empleamos las directivas Input de Angular.

Para efectos de este tutorial, solo consideraremos: el texto del botón, la función click, el contexto del componente padre, si es uppercase (letras en mayúsculas) y svg (nombre de la imágen que se empleará como ícono).
